## 三、已修复 & 待修项

### ✅ 本轮已修复
1. **记录抽屉编辑回显被清空**：原 `watch(typeId)` 会在打开「编辑记录」时把已回填的字段清空，用户看到空白表单。已改为「类型切换手动清空、编辑回显不受影响」，重新构建前端并生效。

### 🟡 待修 / 已知缺口（建议下个迭代处理）
1. **自定义类型未传 schema**：`TypeManage` 新增类型只传 `key/label/icon`，未传字段 schema；后端 `types/create` 接受 `schema ?? null`。结果：自定义类型（如「身高」）记录无字段可填。属 PRD M4（进阶）范围，MVP 可先仅占位，但新增类型当前无实际录入价值。→ 建议 M4 补 schema 编辑表单。
2. **图片 URL 直接拼 `v-html`**：`EventDetail.renderPayload` 的 image 分支把 `obj.url` 未经转义拼入 `v-html`，本地自用风险低，但属潜在 XSS。→ 建议统一 `escapeHtml`。
3. **`/api/topics/search` 前端未接**：PRD FR-18 的「主题内按标签搜索」目前由 `events.list` 的 `tags` 参数实现（`EventTimeline` 顶部筛选），`topics.search` 接口未被前端调用，属冗余。→ 可删除或后续用于跨主题搜索。
4. **移动端 Vant 未接入**：PRD 第 6 项决策为「桌面 Naive UI + 移动 H5 Vant」，目前全站用 Naive UI（靠响应式适配窄屏），未做 Vant 分支。→ MVP 可接受，正式移动端需按决策接入。
5. **前端产物偏大**：Naive UI 全量引入，构建 JS ~1.5MB。→ 后续可改按需引入或分包。
6. **无自动化测试 / 无登录鉴权**：MVP 按决策后端固定 `user_id=1`、无鉴权，符合预期；多用户切换需 M 后续阶段补登录与令牌。
