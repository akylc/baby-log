## 排序拖拽体验修复（3 轮迭代）

## 修复：iOS 上内容不多仍出现整页滚动条
- **根因**：`App.vue` 的 `.app-shell` 用 `height: 100vh`。iOS Safari 的 `100vh` = 大视口高度（含地址栏背后区域），比真实可见区高约 44–83px；而全局 `html,body,#app { height:100% }` 跟随的是小视口，于是 `.app-shell` 溢出 `#app`/body，整页出现滚动条（`.app-shell` 自身的 `overflow:hidden` 挡不住它溢出父级）。
- **修复**：① `.app-shell` 改 `height: 100vh; height: 100dvh;`（100dvh=动态视口=可见区，地址栏伸缩自动贴合；100vh 作旧浏览器兜底）；② 全局 `html,body,#app` 加 `overflow: hidden` 双保险，整页不再滚动，页面内滚动仍由 `.page-area { overflow-y:auto }` 承载。

## iOS 状态栏颜色与 navbar 一致

## 首页列表 keep-alive（返回不闪空 + 返回即刷新）
- **目标**：从其他页面（记录/宝宝）返回首页时，列表不空白闪一下，同时拉取最新数据。
- **方案**：`App.vue` 的 `<router-view>` 用 `<keep-alive include="Home">` 包裹（只缓存首页，记录/宝宝页不缓存，避免表单状态残留）。`Home.vue` 加 `defineOptions({ name: 'Home' })` 供 include 精确匹配；`onMounted(refresh)` 首屏加载，`onActivated` 在返回时触发 `refresh`（用 `activatedOnce` 跳过首次挂载的那次重复加载）；`refresh` 仅在 finally 切 loading，列表数据在请求返回前保持旧值，故不闪空。
- **连带修复 reveal 兼容**：`useRevealRefresh` 原只在 `onMounted` 注册、`onUnmounted` 注销——keep-alive 下组件不卸载，监听会常驻导致所有缓存页回前台一起刷新。改为同时用 `onActivated(add)` / `onDeactivated(remove)`（保持同引用幂等），使监听只在「当前激活的页面」生效；非 keep-alive 页（记录/宝宝）`onActivated` 不触发，行为不变。
