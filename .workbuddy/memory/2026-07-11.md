## 修复登录/注册 no such column: username（第十七轮）
- 现象：前端登录/注册报错 `no such column: username`。
- 根因：磁盘库 `backend/data/momentlog.db` 的 users 表是旧结构 `["id","name","created_at"]`（开发早期建表），而美柚风格重构后的新 schema 为 `id,username,password_hash,salt`；`CREATE TABLE IF NOT EXISTS` 不会给已存在的表补列，所以一碰 username 就报错。
- 修复：在 `backend/src/db/index.ts` 新增 users 表向前兼容迁移（参照已有 feedings 迁移写法）：缺 username 时 `ALTER TABLE users ADD COLUMN username/password_hash/salt` 并用原 name 填充旧行，再 `DROP COLUMN name` 清理废弃列。`tsup` 重建后端、重启（PID 309）。
- 验证：注册 testuser → code 0 返回 token；重复注册→code 1「用户名已存在」；正确密码登录 code 0；错误密码 code 1「用户名或密码错误」。迁移后 PRAGMA 确认 users 列=`[id,created_at,username,password_hash,salt]`（2 行：旧 id=1 + 新注册 id=2）。
- 教训：SQLite + `CREATE TABLE IF NOT EXISTS` 模式下，表结构演进必须写显式迁移（ALTER TABLE），不能靠改建表 DDL 自动生效；旧库已有表不会变。
