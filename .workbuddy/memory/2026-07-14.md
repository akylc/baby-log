## 首页「距上次睡眠」计算优化
- 用户需求：把"距上次睡眠"改为「当前开始睡眠时间(sleep_start) 到 上次醒来时间(sleep_end)」计算。
- 原逻辑：每条睡眠项 `anchorTs = sleepAnchor(s)`（优先醒来时间），"距上次" = `当前 anchorTs − 上一条 anchorTs`，即「当前醒来 − 上次醒来」，不符合预期。
- 改动（frontend/src/views/Home.vue）：
  - 给 timeline item 增加 `gapStartTs` 字段（类型 + items/ref 两处类型声明）。
  - `sleeps` 分支 `gapStartTs = sleep_start`（无则回退 sleepAnchor）；`feedings`/`diapers` 分支 `gapStartTs = anchorTs`（保持原行为）。
  - `buildTimeline` 中「距上次」改用 `it.gapStartTs`（睡眠=开始入睡时间）作为当前端起点；「距现在」仍用 `it.anchorTs`（睡眠=醒来时间，表示醒来到现在多久）。
  - 「距上次」上一条边界 `olderTs` 对睡眠本就是上一条 `sleepAnchor`（醒来时间），跨天边界 `prevDayLastByType['sleep']` 同样用 `sleepAnchor` → 均为「上次醒来时间」。故结果正确为「本次入睡 − 上次醒来」（两次睡眠间的清醒时长）。
- 验证：`pnpm build` 通过；运行实例 `:26712` 已实时读盘提供新前端 `index-CeBim-ZW.js`（HTML 禁缓存，刷新即生效）。

## iOS 弹窗滑动导致背景页面滚动穿透修复
- 用户需求：iOS 上滑动「筛选记录类型」等底部抽屉弹框时，背景页面（.page-area）会一起滚动，需优化。
- 根因：`.sheet-mask` 是 `position:fixed` 全屏遮罩，但 iOS 仍把 touch 滚动路由到背景滚动容器 `.page-area`（`App.vue` 的 `overflow-y:auto`），造成 scroll chaining 穿透。
- 改动：
  1. `App.vue`：`.page-area` 增加 `.scroll-locked{overflow:hidden}`（scoped 命中背景容器）。
  2. `Home.vue`：导入 `onUnmounted`；新增 `pageAreaEl` + `syncScrollLock()`，在 `onMounted` 及 `watch([filterShow,switchShow,editShow])` 时给 `.page-area` 加/去 `scroll-locked`（任一弹窗打开即锁背景，全关即解锁）；`onUnmounted` 清理。
  3. `Home.vue` 样式：`.sheet-body` 加 `overscroll-behavior-y:contain`，防止弹窗自身滚到边界再滑穿透到背景（双保险）。
- 验证：`pnpm build` 通过；产物 CSS 含 `.page-area.scroll-locked{overflow:hidden}` 与 `.sheet-body{…overscroll-behavior-y:contain}`；运行实例 :26712 已实时读盘提供新 CSS（刷新即见）。
- 状态：已随发版提交推送（见下方「发版 v0.0.13」）。改动文件：frontend/src/App.vue、frontend/src/views/Home.vue。
